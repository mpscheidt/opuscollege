Whitepaper Web-based Init-Wizard


Objective:

At the first time the user starts opusCollege in the webbrowser, ask the user for essential configuration settings


Issues:

  • Initialize database connection settings

  • Setup the database (if not yet existing)

  • Create (administrative) user for opusCollege


Initialize database connection settings

Advantage: Avoid to have jdbc settings in files, allow for individual (secure) passwords.

Affected settings:

  • JDBC connection string – Default: jdbc:postgresql://localhost:5432/opusCollege

  • username – Default: postgres

  • password – No default



JDBC connection settings are different for each installation. In many cases the database will reside on the same host as the tomcat server but this might not always be the case.

For a secure setup the password has to be chosen at each site individually.
